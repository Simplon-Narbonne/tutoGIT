Première étape : Générer une clef SSH.

Une clef SSH est un couple de fichiers générés sur votre machine, une clef privée et une clef publique, servant à vous identifier sur différents sites internet sans utiliser vos logins/mots de passe.

Pour générer une clef SSH ouvrez un terminal (ctrl+alt+t sur toutes les bonnes distros) et tapez la commande : ssh-keygen

Le terminal vous invite a entrez un nom de fichier et une passphrase, ce n'est pas obligatoire.

Si vous n'entrez pas de nom de fichier, le nom par defaut est : id_rsa pour la clef privée et id_rsa.pub pour la clef publique. Si vous entrez une passphrase, vous devrez la taper à chaque ouverture de connexion.

Deuxième étape : Ajouter une clef SSH à github.

Dans votre terminal entrez cat ~/.ssh/id_rsa.pub pour faire apparaître votre clef publique et copiez la. Allez dans les paramètres de votre compte github puis sur l'onglet SSH KEYS. Cliquez sur ajouter une clef, donnez lui un titre puis collez votre clef et validez.

Lorsque vous créez un dépôt, activez bien le protocole ssh à la place du protocole https sous la ligne QUICK SETUP (vos liens doivent apparaître sous cette forme git@github.com:votrecompte/nomdudepot.git)

/!\ Ne donnez jamais votre clef privée, ne la partagez pas! Elle est aussi importante que votre mot de passe /!\
